Algorithm Comparison:
During testing, all algorithms returned 0.000000s. This is because the array size (100) is too small for the CPU to measure in seconds.

Analysis:

Linear Search: O(n) - Slowest because it scans one by one.

Binary Search: O(log n) - Faster but requires sorting first.

Hashing: O(1) - Fastest because it uses direct indexing.

Conclusion:
Hashing is the most efficient. Even though we got 0.000000s across the board, Hashing stays that fast even when the data grows, while Linear Search will eventually slow down.
